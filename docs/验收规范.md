## 0. 阅读须知（agent 执行约束）

1. 本文档每一项验收点都标注 **【PASS 标准】** 与 **【检查方法】**，agent 执行时须实际操作或运行命令，禁止仅凭代码静态推断结论。
2. 涉及具体 URL/端口/密码的检查项，须使用本文档第 9 章给定的部署参数执行。
3. 命令前缀约定：本地执行用 `curl`/`psql`/`mysql`/`redis-cli`；远程执行前加 `ssh` 跳板（见 §9.1）。
4. 每项验收结果只能是 **PASS / FAIL / BLOCKED** 三态之一，禁止用"基本通过"等模糊表述。
5. 发现 FAIL 项时，按 §10.2 流程提 issue，**禁止擅自修改本文档的判定标准**。
6. 遇到契约不一致（见 §8.0）时，验收暂停，待团队对齐后再恢复。

确认"食愈校园"项目 P0–P3 全部功能、非功能指标、三端兼容、部署运维符合设计要求，达到可上线状态。

- 基础设施：MySQL、PostgreSQL、Redis、RabbitMQ、Nacos、阿里 OSS
- 服务器：118.178.229.21

```
开发完成 → 自测（开发者）→ 联调验收（前后端联调人）→ 上线验收（运维 + 验收 agent）
```

每个阶段产出对应的验收报告（模板见 §10.3），全部 PASS 后进入下一阶段。

## 2. 验收流程

| 阶段 | 验收人 | 验收方式 | 通过门槛 |

### 3.1 【P0 必做】拍照识菜（异步轮询 UI）

| 验收点 | 检查方法 | PASS 标准 |
|---|---|---|
| 拍照 tab 入口 | 打开"添加菜品"弹窗，确认 Tab 行含"📷 拍照" | Tab 存在且可点击切换，未选中态不显示拍照区 |
| 后置相机调起 | 移动端真机点拍照 tab → 点上传区 → 选相机 | `<input>` 含 `capture="environment"`，移动端调起后置相机 |
| 图片预压缩 | DevTools Network 上传请求 payload 检查 | 上传图片 JPEG 长边 ≤ 800px，体积 ≤ 200KB（用 `< 200KB` 严格判定）|
| 异步任务提交 | 选图后看 Network：`POST /api/recognize-dish` | 请求为 multipart/form-data，响应 < 200ms 内返回 `taskId`/`task_id` |
| 进度条显示 | 提交后弹窗显示 loading-ring + 进度条 | 进度条存在，0→80% 前 3 秒线性增长，"AI 正在认菜..."文案显示 |
| 轮询调度 | DevTools Network 看 `GET /api/recognize/result/{taskId}` | 间隔约 1~1.5s，最多 30 次，30s 内必须有终态（done/failed）|
| 30s 超时处理 | 故意制造后端不返回（断 MQ 或 mock pending）| 30s 后 UI 显示"识别超时"提示 + 重试按钮，不卡死 |
| 识别结果可编辑 | 一次成功识别后看结果区 | 显示缩略图 + 表单（name/category/kcal/protein/carb/fat/unit），所有字段可编辑 |
| 低置信度提示 | 一次 confidence < 0.6 的识别结果 | 表单标题显示"🤔 不太确定，请核对" |
| 确认入盘 | 点"确认添加到菜盘" | 调用 `addFood()`，菜盘对应餐次出现该菜品，toast"✓ 已添加：xxx" |
| 取消重置 | 识别中点"取消" | 轮询停止（Network 不再有 GET 请求），状态清空，回到上传区 |
| 失败重试 | 后端返回 status=failed | UI 显示"识别失败：xxx" + 重试按钮，点重试重新走流程 |
| 跨 tab 隔离 | 拍照识别中切到搜索 tab 再切回 | 拍照进度保留，不互相干扰 |

## 附录 A：验收决策摘要（关键判定）

1. **契约一致性是前置阻断项**（§8.0）：联调开始前 16 项不一致必须先对齐，避免返工。
2. **30 秒识图超时是硬约束**：轮询次数/间隔可议，但用户总等待不超过 30 秒。
3. **首 token < 2 秒、识图 POST < 200ms、推荐 < 50ms** 为性能红线，不达标视为 FAIL。
4. **uv 项目管理**：必须 `uv sync` 一键复现 + `uv.lock` 提交，不允许保留 `requirements.txt`。
5. **服务器密码 1234 全量统一**：Redis/MySQL/Pg/RabbitMQ/Nacos，部署连接串必须含此密码。
6. **降级必须实现**：SSE 降级、识图失败重试、后端不通 fallback 到本地行为，任一缺失视为不通过。
