## 1. 统一原则

### 1.1 字段命名：全 snake_case

- 所有 JSON 字段、URL query 参数、Pydantic 模型字段、JS 对象 key 一律 `snake_case`
- 后端 Pydantic 默认即 snake_case，无需 `alias` 转换
- 前端 JS 调用时不强制转 camelCase，**保持与后端一致**，省去转换层
- 数据库列名沿用 snake_case（MySQL / PostgreSQL 原生支持）
- Redis key 沿用 `namespace:detail` 风格（如 `chat:ctx:{user_id}`）

**例外**：HTTP header（如 `Authorization`、`Content-Type`、`X-Request-Id`）沿用 HTTP 标准 Pascal-Case。

### 1.2 接口路径：RESTful 风格

- 资源用名词复数：`/api/dishes`、`/api/sport/records`
- 子资源用嵌套：`/api/dishes/{id}/reviews`、`/api/health/profile`
- 路径参数用 snake_case 单数：`{task_id}`、`{user_id}`、`{dish_id}`
- 所有路径前缀 `/api`，B 端前缀 `/api/admin`
- 时间相关 query 参数：`date=YYYY-MM-DD`、`week=YYYY-Wxx`

### 1.3 请求方法：按语义

| 方法 | 语义 | 本项目用法 |
|---|---|---|
| GET | 查询（无副作用） | 列表、详情、轮询、菜单、榜单 |
| POST | 创建 / 复杂查询 / 触发动作 | 提交识图、发起对话、创建评价、加入拼饭 |
| PUT | 整体更新 | （暂未使用） |
| PATCH | 局部更新 | 偏好局部修改（可选） |
| DELETE | 删除 | 取消拼饭、隐藏评价（admin） |

**复杂查询例外**：周报虽是"查询"，但因后端从 PostgreSQL 聚合 + AI 生成，用 GET 携带 `user_id` + `week` query 即可（后端不依赖前端传 7 天数据）。

### 1.4 统一响应格式

```json
{
  "code": 0,
  "message": "ok",
  "data": { /* 业务字段，缺失时返回 {} 不返回 null */ }
}
```

- `code === 0` 成功，非 0 见错误码表（§5）
- `message` 面向用户可读，前端可直接 toast
- HTTP 状态码仍按 REST 规范（200/4xx/5xx）
- SSE 流式响应不套该格式，按事件流（§3.3）
- 所有时间戳 ISO8601 带时区 `+08:00`

### 1.5 鉴权

- C 端：`Authorization: Bearer {anon_token}`（首次访问后端下发，存 localStorage）
- B 端：`Authorization: Bearer {admin_token}`（口令登录获取）
- `user_id` 由后端从 token 解析，**前端请求体也带 `user_id` 用于无 token 兜底**

## 2. 16 处不一致统一决策摘要

| # | 不一致点 | 原冲突 | 统一为 |
|---|---|---|---|
| 1 | 识图任务 ID 字段名 | `taskId` (BE) vs `task_id` (FE) | `task_id` |
| 2 | 识图任务 ID 值前缀 | `task_8f2a...` (BE) vs `rec_xxx` (FE) | `rec_` 前缀（值，非字段名） |
| 3 | 识图结果字段名 | `result`+全字段 (BE) vs `dish`+仅 confidence (FE) | `dish`，**含 confidence / matched / alternatives 全字段** |
| 4 | 识图响应 progress 字段 | 无 (BE) vs `progress: 30` (FE) | **后端加** `progress` 字段（0-100 百分比） |
| 5 | 轮询参数 | 500ms 起 + 1s 间隔 / 30 次 (BE) vs 1.5s 间隔 / 20 次 (FE) | **1.5s 间隔 / 20 次**（30s 硬约束正好达标） |
| 6 | AI 对话请求体 | `{user_id, message, mode, tray, nutrition}` (BE) vs `{persona, messages, context:{food_tray, mode, nutrition, mode_target}}` (FE) | **合并**：`{user_id, persona, messages, context:{food_tray, mode, nutrition, mode_target}}`，`user_habits` 后端基于 `user_id` 自取 |
| 7 | 人设 ID 枚举 | `canteen_aunt/senior_brother/senior_sister` (BE) vs `auntie/senior_bro/senior_sis` (FE) | `canteen_aunt / senior_brother / senior_sister` |
| 8 | 周报接口方法 | GET (BE) vs POST 带 7 天数据 (FE) | **GET** `/api/weekly-report?user_id=&week=`，后端从 PostgreSQL 取数据，前端不传 7 天 body |
| 9 | 健康档案路径 | `/api/health/profile` (BE) vs `/api/health-profile` (FE) | `/api/health/profile` |
| 10 | 运动路径 | `/api/sport/record` (BE) vs `/api/exercise` (FE) | `/api/sport/records`（POST 创建 / GET 列表） |
| 11 | 情绪响应字段 | `{id, user_id, mood, note, created_at, ai_comfort}` (BE) vs `{healing_text, suggestion?}` (FE) | **合并**：`{id, user_id, mood, note, created_at, ai_comfort, suggestion?}` |
| 12 | 推荐响应 | `{gaps, suggestions:[{...dish, reason}]}` (BE) vs `{dish, reason}` (FE) | 后端契约 `{gaps, suggestions:[{...dish, reason}]}`，前端取 `suggestions[0]` |
| 13 | 菜单响应 | `CanteenMenu{breakfast, lunch, dinner}` (BE) vs `{items, recommended}` (FE) | `CanteenMenu{canteen, date, breakfast, lunch, dinner}`，`MenuItem` 含 `is_recommended` 字段 |
| 14 | 拼饭/评价接口 | 后端未明确 | 后端补全：`/api/pinfan` GET/POST、`/api/pinfan/{id}/join` POST、`/api/dishes/{id}/reviews` GET/POST |
| 15 | B 端 admin 接口 | 仅 canteen-stats/dish-heat (BE) | 后端补全：`/api/admin/{overview, dish-heat, unsold, nutrition-stats, menu, reviews, reviews/{id}/reply}` |
| 16 | API 字段命名整体 | Pydantic camelCase (BE) vs 混用 snake_case (FE) | **全 snake_case**，Pydantic 默认即可，前端不转 camelCase |

## 3. 接口清单表（统一契约）

> 所有非 SSE 接口返回 `{code, message, data}`；SSE 按事件流。

### 3.1 P0 识图异步链路

#### 3.1.1 POST /api/recognize-dish — 提交识菜任务

**请求**：`multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| file | File | 是 | 图片 jpg/png/webp，≤ 5MB |
| user_id | string | 否 | 用于忌口过滤 |
| meal_hint | string | 否 | breakfast / lunch / dinner |

**响应**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "task_id": "rec_8f2a3b4c5d6e",
    "status": "pending"
  }
}
```

**错误**：`40001` 文件超限/格式错；`42901` 限流；`50005` OSS 上传失败；`50006` MQ 投递失败。

**响应（done）**：

### 3.2 P0 AI 对话

#### 3.2.1 POST /api/chat — SSE 流式对话

**请求**：`application/json`，`Accept: text/event-stream`

```json
{
  "user_id": "u_demo",
  "persona": "canteen_aunt",
  "messages": [
    { "role": "user", "content": "我蛋白够不够？" },
    { "role": "assistant", "content": "你今天吃了鸡胸肉..." }
  ],
  "context": {
    "food_tray": [
      { "name": "鸡胸肉", "category": "肉类", "kcal": 130 }
    ],
    "mode": "fitness",
    "nutrition": { "kcal": 130, "protein": 28, "carb": 1, "fat": 3 },
    "mode_target": { "kcal": 1800, "protein": 110, "carb": 220, "fat": 60 }
  }
}
```

**说明**：
- `user_id`：用于后端取 `user_habits`（长期记忆）+ Redis 短期记忆
- `persona`：当前人设 id（与 `user_habits.ai_personality` 同步，前端切换即时传新值）
- `messages`：最近 6 条对话（user / assistant 交替），后端可作为 fallback（主取 Redis）
- `context`：当前菜盘摘要 + 模式 + 营养总计 + 模式目标
- `user_habits`：**前端不传**，后端基于 `user_id` 自取

**响应**（流式 `text/event-stream`）：

```
event: delta
data: {"delta":"蛋白","done":false}

event: delta
data: {"delta":"已经","done":false}

event: memory_hint
data: {"hint":"你上周说胃不舒服"}

event: delta
data: {"delta":"还吃红烧鸡？","done":false}

event: heartbeat
data: {"type":"heartbeat"}

event: done
data: {"delta":"","done":true}
```

**降级**：客户端 `Accept: application/json` 或加 `?stream=false` → 返回完整 JSON：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "text": "你今天蛋白 28g，离健身目标 110g 还差不少……",
    "memory_hint": "你上周说胃不舒服"
  }
}
```

**错误**：`50003` 模型调用失败；`42901` 限流。

### 3.3 P1 推荐 / 周报 / 偏好 / 多模态 / 分享

#### 3.3.1 GET /api/recommend — 智能推荐补菜

**请求**：`GET /api/recommend?user_id=u_demo&mode=fitness&tray=%5B%7B%22id%22%3A%22f108%22%7D%5D`

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| user_id | string | 是 | 取 user_habits 过敏过滤 |
| mode | string | 是 | daily / fitness / weight_loss |
| tray | JSON | 是 | URL-encoded `[{id, name, category, kcal, protein, carb, fat}]` |

**响应**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "gaps": ["蛋白质缺口 60g", "午餐缺蔬菜"],
    "suggestions": [
      {
        "id": "f108",
        "name": "鸡胸肉",
        "category": "肉类",
        "unit": "份(100g)",
        "kcal": 130,
        "protein": 28,
        "carb": 1,
        "fat": 3,
        "reason": "高蛋白低脂，补蛋白缺口"
      }
    ]
  }
}
```

> 推荐算法**不调 AI**，纯本地计算，延迟 < 50ms。前端取 `suggestions[0]` 渲染推荐卡。

  "steps": 8500,

## 8. 时间格式与 ID 规范

### 8.1 时间格式

- 所有时间戳 ISO8601 带时区：`2026-07-03T14:30:00+08:00`
- 日期：`YYYY-MM-DD`（如 `2026-07-03`）
- 周次：`YYYY-Wxx`（如 `2026-W27`）
- 时间：`HH:mm`（如 `12:00`）

| `s` + 时间戳 | 外卖截图 | `s001` |
| `m` + 数字 | 食堂菜单引用 | `m001` |

## 10. SSE 事件流规范

### 10.1 事件类型

| event | data 内容 | 说明 |
|---|---|---|
| `delta` | `{"delta":"...","done":false}` | 流式 token |
| `memory_hint` | `{"hint":"..."}` | 记忆提示 |
| `heartbeat` | `{"type":"heartbeat"}` | 心跳保活（每 15s） |
| `done` | `{"delta":"","done":true}` | 流结束 |
| `error` | `{"code":xxx,"message":"..."}` | 流中错误 |

### 10.2 响应头

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no
```

### 10.3 降级

- 客户端 `Accept: application/json` → 返回完整 JSON
- 客户端加 `?stream=false` → 同上
- nginx 必须配 `proxy_buffering off;`
