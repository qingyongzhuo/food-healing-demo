# 食愈校园 — 前端技术规范

> 版本：v1.0
> 适用范围：`food-healing-demo/` 全部前端代码（C 端 `index.html` + B 端 `admin.html`）
> 配套文档：`docs/FRONTEND_DESIGN.md`（V2 设计）
> 量级：单校单食堂 <1 万用户；P0–P3 全做；三端兼容（手机 / 平板 / 网页）

---

## 0. 如何使用本文档

- 本规范是**评审清单**，每条 PR 必须对照检查；不是教程。
- 凡标注【必须】的条款不得破例；标注【建议】的可在评审中讨论。
- 现有代码（`index.html` / `js/main.js` / `css/style.css`）作为正例引用，新代码必须沿用其风格。
- 性能改动遵循最小改动原则：**先并发度 → 再缓存 → 再批处理**，不轻易引入新依赖。

## 1. 总则

### 1.1 规范目的

- 统一团队风格，避免 P0–P3 十余功能上线后代码不可维护。
- 锁死技术栈版本，防止 CDN 引入构建工具破坏纯静态部署。
- 保留 V1 治愈系视觉与手机外壳身份感，新组件不得另起体系。

### 1.2 适用范围

- C 端：`index.html` + `js/`（除 `js/admin/`）+ `css/`（除 `css/admin.css`）。
- B 端：`admin.html` + `js/admin/` + `css/admin.css`。B 端独立但共用 `js/api.js` / `js/store.js` / 治愈系色板。

### 1.3 技术栈版本固定【必须】

| 类目 | 固定版本 / 来源 | 不得替换 |
|---|---|---|
| Vue | `https://unpkg.com/vue@3/dist/vue.global.prod.js` | SFC / Vite / Pinia / Vuex |
| Tailwind | `https://cdn.tailwindcss.com` | 自建 PostCSS |
| Vue Router（P1 起） | `https://unpkg.com/vue-router@4`，**hash 模式** | history 模式（CDN 静态托管刷新 404） |
| ECharts（仅周报 / B 端） | `https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js` | 不在 `index.html` 顶部 preload，按需动态加载 |
| TypeScript | 不引入 | 用 JSDoc 风格类型注释替代（见 §3.3） |

### 1.4 设计原则【必须】

1. CDN 模式不引构建工具：所有 JS `<script src>` 顺序加载，组件用 Vue3 全局 `app.component()` 或挂 `window`。
2. 渐进增强：后端不通时降级到本地行为（AI 文案回退 `generateAiText()`），**不能白屏**。
3. 性能优先级：并发 → 缓存 → 批处理。识图、AI 对话、推荐三者可并行触发，不串行 `await`。
4. B 端破例：`admin.html` 可破例桌面布局，但**色板 / 字体 / 圆角 / 虚线**仍沿用治愈系。

### 1.5 非目标【必须不做】

- 不引入 Vue SFC / Vite / Webpack / Rollup。
- 不引入 Pinia / Vuex（用 `reactive` 自建 store）。
- 不引入 TypeScript。
- 不替换现有 `matcher()` 餐次匹配算法。
- 不做游戏化养成系统。

### 2.2 拆分原则【必须】

| 文件 | 装什么 | 不装什么 |
|---|---|---|
| `constants.js` | 数据 / 配置常量（`FOODS_DATA` / `CATEGORIES` / `ICONS` / `MODES` / `STRUCTURES` / `CATEGORY_META` / `MODE_COLORS` / `PERSONAS`） | 函数 |

| `api.js` | 网络请求封装 | 业务逻辑判断 |

| `main.js` | `createApp` + `app.component(...)` 注册 + `mount` | 大段业务函数 |

### 2.3 脚本加载顺序【必须】

`index.html` 底部按依赖顺序加载，顺序敏感：

```html
<!-- 1. 第三方 -->
<script src="https://unpkg.com/vue@3/dist/vue.global.prod.js"></script>
<script src="https://cdn.tailwindcss.com"></script>

<!-- 2. 自有逻辑（顺序敏感：constants → utils → store → api） -->
<script src="js/constants.js"></script>
<script src="js/utils.js"></script>
<script src="js/store.js"></script>
<script src="js/api.js"></script>

<!-- 3. 组件（依赖 store + api） -->
<script src="js/components/persona-picker.js"></script>
<script src="js/components/add-food-modal.js"></script>
<script src="js/components/ai-chat-panel.js"></script>
<!-- ... -->

<!-- 4. 入口（最后加载，注册所有组件并 mount） -->
<script src="js/main.js"></script>
```

`admin.html` 加载顺序类似，但加载 `js/admin/` 下脚本，不加载 C 端组件。

### 2.4 迁移步骤【建议】

1. 先把 `FOODS_DATA / CATEGORIES / ICONS / MODES / STRUCTURES / CATEGORY_META / MODE_COLORS` 抽到 `constants.js`。
2. 把 `calcNutrition / matcher / generateAiText / getBarColor` 抽到 `utils.js`。
3. 新建 `store.js` / `api.js`。
4. 新功能直接以组件形式写在新文件，不再往 `main.js` 塞。
5. 现有 `index.html` 内联模板可暂不拆，P0 完成后再拆 `AddFoodModal` / `AiChatPanel` / `PersonaPicker`。
6. 不重写已有功能代码（`matcher` / `generateAiText` / 营养环保持不动）。

## 3. 代码规范

### 3.1 命名规范【必须】

#### 3.1.1 变量 / 函数

- **camelCase**，动词 + 名词，见名知意。
- 正例（现有代码）：`addFood` / `removeFood` / `switchMode` / `refreshMeal` / `closeModal` / `saveState` / `selectFilterCategory` / `showToast` / `clearTray` / `calcNutrition` / `generateAiText` / `getBarColor` / `categoryEmoji` / `categoryColor`。
- 反例：`add1` / `doStuff` / `tmp` / `data2`。

```javascript
// 正例
function addFood(food) { /* ... */ }
const recognizeTaskId = ref(null);
const modalTab = ref('search');

// 反例
function add(f) { /* ... */ }
const tid = ref(null);
const mt = ref('search');
```

#### 3.1.2 ref / reactive 命名

- ref 用 camelCase，布尔值加 `is/has/can` 前缀：`isTyping` / `isCelebrate` / `showModal` / `hasAlternatives` / `chatSending`。
- reactive store 用 `xxxStore` 后缀：`trayStore` / `chatStore` / `recognizeStore` / `prefStore` / `socialStore` / `cacheStore` / `adminStore`。

#### 3.1.3 组件命名

- **PascalCase**，至少两个单词，避免与 HTML 原生标签冲突。
- 正例：`AddFoodModal` / `AiChatPanel` / `PersonaPicker` / `RecommendCard` / `NutritionRing` / `WeeklyReportPage` / `SocialModule`。
- 反例：`Modal` / `Card` / `Add`。

#### 3.1.4 文件命名

- `kebab-case`：`add-food-modal.js` / `ai-chat-panel.js` / `recommend-card.js`。
- 文件名与组件名一一对应：`add-food-modal.js` → `AddFoodModal`。

#### 3.1.5 CSS class 命名

- BEM-like 风格：`block` / `block__element` / `block--modifier`。
- 正例（现有代码）：`.meal-card` / `.meal-card__header` / `.meal-card--breakfast`；`.food-tag` / `.food-tag__name` / `.food-tag--active`。
- 现有代码未严格用 BEM（如 `.meal-card .meal-header`），**新代码必须用 BEM**，旧代码不强制重构。
- 工具类用 Tailwind（`flex` / `gap-2` / `text-sm`），业务类用自定义 CSS。

```css
/* 正例（新代码） */
.recommend-card { /* ... */ }
.recommend-card__title { /* ... */ }
.recommend-card__btn--disabled { /* ... */ }

/* 反例 */
.recommendCardTitle { /* ... */ }
.recommend_card_title { /* ... */ }
```

#### 3.1.6 常量命名

- 全大写 + 下划线：`FOODS_DATA` / `CATEGORIES` / `ICONS` / `MODES` / `STRUCTURES` / `CATEGORY_META` / `MODE_COLORS` / `PERSONAS`。

### 3.2 Vue3 setup 写法规范【必须】

#### 3.2.1 入口写法

沿用现有 `js/main.js` 写法：

```javascript
const { createApp, ref, computed, watch, onMounted, onUnmounted, nextTick } = Vue;

createApp({
  setup() {
    // 1. 状态
    const currentMode = ref(localStorage.getItem('currentMode') || 'daily');
    const foodTray = ref(JSON.parse(localStorage.getItem('foodTray') || '[]'));

    // 2. computed
    const nutrition = computed(() => calcNutrition(effectivePlan.value));

    // 3. 方法
    function addFood(food) { /* ... */ }

    // 4. watch
    watch(nutrition, (newVal) => { /* ... */ }, { deep: true });

    // 5. 生命周期
    onMounted(() => { /* ... */ });
    onUnmounted(() => { /* ... */ });

    // 6. return（模板用到的全部暴露）
    return {
      currentMode, foodTray, nutrition,
      addFood,
    };
  }
}).mount('#app');
```

#### 3.2.2 规则

- 不用 Options API（`data()` / `methods` / `computed` 选项式），统一 Composition API + `setup()`。
- ref 在模板中**不写 `.value`**，在 JS 中**必须写 `.value`**。
- 计时器 / 监听器在 `onUnmounted` 中清理（现有代码已遵守，见 `main.js` 第 612–617 行）。
- 同一 setup 内 ref 数量超过 30 时必须拆组件或下沉 store。
- `watch` 默认浅监听，需要深监听加 `{ deep: true }`（现有 `watch(nutrition, ..., { deep: true })` 为正例）。

### 3.3 JS 规范（CDN 模式无 TS，用 JSDoc 风格类型注释）【必须】

#### 3.3.1 JSDoc 类型注释

CDN 模式不引 TypeScript，但函数签名必须用 JSDoc 标注参数与返回类型，便于 IDE 提示与评审：

```javascript
/**
 * 添加菜品到菜盘
 * @param {Food} food - 菜品对象
 * @returns {void}
 */
function addFood(food) {
  const trayId = 't' + Date.now() + Math.random().toString(36).substr(2, 5);
  foodTray.value.push({ ...food, trayId, isCustom: false });
  saveState();
}

/**
 * 食物对象
 * @typedef {Object} Food
 * @property {string} id - 菜品 id（f001 / custom_xxx / r001 / v001 / s001 / m001 / rec_xxx）
 * @property {string} name - 菜名
 * @property {'主食'|'肉类'|'蔬菜'|'蛋奶'|'汤品'|'水果'} category - 分类
 * @property {string} unit - 份量描述
 * @property {number} kcal - 热量
 * @property {number} protein - 蛋白质 g
 * @property {number} carb - 碳水 g
 * @property {number} fat - 脂肪 g
 */

/**
 * 异步轮询识菜
 * @param {File} file - 图片文件
 * @param {{onProgress?: (n: number) => void, onStatus?: (s: string) => void}} [opts]
 * @returns {Promise<Food>}
 */
async function recognizeWithPolling(file, opts) { /* ... */ }
```

#### 3.3.2 其他 JS 规则

- 字符串拼接用户输入前必须转义（防 XSS，见 §13）。
- `const` 优先，仅在确实需要重赋值时用 `let`，禁用 `var`。
- 解构赋值优先（`const { createApp, ref } = Vue;` 为正例）。
- 模板字符串优先于字符串拼接：`` `已添加：${name}` ``，不用 `'已添加：' + name`。
- 数字字面量含小数时，比较用 `Math.abs(a - b) < 1e-6` 而非 `a === b`。
- 异步函数必须 try/catch，错误经 `ApiError` 抛出由调用方处理（见 §5）。

### 3.4 CSS 规范【必须】

#### 3.4.1 CSS 变量优先

所有颜色 / 圆角 / 间距 / 阴影**必须用 CSS 变量**，禁止硬编码：

```css
/* 正例 */
.recommend-card {
  background: var(--color-card);
  border: 2px dashed var(--color-dashed);
  border-radius: var(--radius-lg);
  padding: var(--spacing-md);
  box-shadow: var(--shadow-sm);
}

/* 反例 */
.recommend-card {
  background: #FFFFFF;
  border: 2px dashed #E8D5B7;
  border-radius: 16px;
  padding: 16px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.04);
}
```

现有 `:root` 变量见 `css/style.css` 第 1–30 行。新组件如需新色，先在 `:root` 加变量，再使用。

#### 3.4.2 禁用 !important

- **禁止使用 `!important`**，唯一例外是 `prefers-reduced-motion` 兜底（现有代码已遵守，见 `css/style.css` 第 891–897 行）。

#### 3.4.3 选择器层级

- 最多 3 层嵌套：`.meal-card .meal-header .meal-title` 为正例。
- 禁止 `#id` 选择器（仅 `#app` / `#modal-title` 等 ARIA 引用例外）。
- 禁止 `*` 通配（基础重置 `* { margin: 0; }` 例外）。

#### 3.4.4 顺序

属性顺序：定位 → 盒模型 → 排版 → 视觉 → 动画。现有 `.ai-bubble`（`css/style.css` 第 333–347 行）为正例。

#### 3.4.5 单位

- 间距 / 圆角 / 字号用 px（CDN 模式无 rem 转换）。
- 透明度用 `rgba()` 或 `color + '20'`（现有 `categoryColor(item.category) + '15'` 为正例，代表 15% 透明度）。
- 媒体查询断点用 px（见 §8）。

## 4. 状态管理规范

### 4.1 store.js 拆分【必须】

用 Vue3 `reactive` 自建轻量 store，按域切分（不引 Pinia / Vuex）：

```javascript
// js/store.js
const { reactive, ref } = Vue;

// 菜盘域（持久化）
const trayStore = reactive({
  foodTray: JSON.parse(localStorage.getItem('foodTray') || '[]'),
  currentMode: localStorage.getItem('currentMode') || 'daily',
  save() {
    localStorage.setItem('foodTray', JSON.stringify(this.foodTray));
    localStorage.setItem('currentMode', this.currentMode);
  }
});

// AI 对话域（仅内存）
const chatStore = reactive({
  messages: [],
  sending: false,
  persona: localStorage.getItem('persona') || '',
});

// 识菜域（仅内存）
const recognizeStore = reactive({
  preview: null,
  taskId: null,
  progress: 0,
  statusText: '',
  result: null,
  error: '',
});

// 用户偏好域（持久化 + 同步 user_habits）
const prefStore = reactive({
  allergens: JSON.parse(localStorage.getItem('allergens') || '[]'),
  healthProfile: JSON.parse(localStorage.getItem('health_profile') || '{}'),
  save() { /* localStorage 写回 + 调 api.savePreferences 同步 */ }
});

// 社交域（仅内存，token 持久化）
const socialStore = reactive({
  anonymousToken: localStorage.getItem('anon_token') || '',
  leaderboard: null,
  pinfanList: [],
});

// API 缓存域
const cacheStore = reactive({
  menu: { data: null, expireAt: 0 },
  weeklyReport: { data: null, expireAt: 0 },
});

// B 端 admin 域（admin.html 独立使用）
const adminStore = reactive({
  token: localStorage.getItem('admin_token') || '',
  stats: null,
  dishHeat: [],
  menuDraft: null,
});

// 挂到 window 方便 CDN 模式访问
window.stores = { trayStore, chatStore, recognizeStore, prefStore, socialStore, cacheStore, adminStore };
```

### 4.2 localStorage 持久化策略【必须】

| 数据 | 存储位置 | 期限 | 说明 |
|---|---|---|---|
| `foodTray` / `currentMode` | localStorage | 永久 | 复用现有 key（现有 `main.js` 第 526–528 行已实现） |
| `allergens` / `health_profile` / `persona` | localStorage | 永久 | 新增，本地兜底 |
| `anon_token` / `admin_token` | localStorage | 永久 | 身份 |
| `weekly_history_YYYYMMDD` | localStorage | 30 天 | 每日一条，超 30 条触发清理 |
| `dismissed_rec_YYYYMMDD` | localStorage | 当日 | 推荐关闭记录 |
| `chatMessages` | **仅内存** | - | 关闭页面即清，避免隐私 |
| 识菜临时数据 | **仅内存** | - | 确认后即清 |
| 菜单 / 周报 API 结果 | `cacheStore` | 5 分钟 | TTL 过期重新请求 |
| `user_habits` 明文 | **不存前端** | - | 后端维护（见 §4.4） |

### 4.3 响应式数据流【必须】

```
用户操作 → store.action() → api.xxx() → 后端
                ↓                         ↓
        store.xxx 改变              返回 data 写回 store
                ↓
        watch / computed 自动触发 UI 更新
```

- 组件不直接调 `api.js`，必须经 store action 中转（除纯展示组件）。
- 持久化在 store 的 `save()` 方法内统一处理，不散落在组件里。
- 现有 `saveState()` 函数（`main.js` 第 525–528 行）迁移为 `trayStore.save()`。

### 4.4 user_habits 同步策略【必须】

`user_habits` 是后端维护的用户长期记忆载体，**前端不存明文，不传给 AI 接口**：

| 数据 | 写入时机 | 前端做什么 |
|---|---|---|
| `allergens` | 用户改设置页 | 调 `POST /api/preferences` |
| `health_profile` | 用户改健康档案 | 调 `POST /api/health-profile` |
| `recent_moods` | 用户提交情绪日记 | 调 `POST /api/mood` |
| `recent_nutrition_trends` | 每日 23:00 后端定时任务 | 前端不写 |
| `mentioned_facts` | AI 对话过程中后端抽取 | 前端不写 |
| `persona` | 用户切换人设 | 调 `POST /api/preferences` |

前端只持久化 `allergens` / `healthProfile` / `persona` 用于本地拦截（忌口弹窗）和离线兜底。

### 4.5 兼容现有代码【建议】

迁移策略：现有 `foodTray` / `currentMode` ref 改为从 `trayStore` 引用，所有 `saveState()` 调用改为 `trayStore.save()`。其余 ref 暂保留在 setup 内，**不强制一次性迁移**（遵循最小改动原则）。

## 5. API 调用规范

### 5.1 api.js 统一封装【必须】

所有后端调用必走 `js/api.js`，禁止组件内直接 `fetch`。

```javascript
// js/api.js
const API_BASE = window.API_BASE || 'http://localhost:8000/api';

/**
 * 统一请求函数
 * @param {string} path - 接口路径，如 '/recognize-dish'
 * @param {RequestInit & {admin?: boolean}} [options]
 * @returns {Promise<any>} - 解构后的 data 字段
 */
async function request(path, options = {}) {
  const url = API_BASE + path;
  const headers = { 'Accept': 'application/json', ...(options.headers || {}) };

  // token 预留（user / admin 区分）
  const token = options.admin
    ? localStorage.getItem('admin_token')
    : localStorage.getItem('anon_token');
  if (token) headers['Authorization'] = 'Bearer ' + token;

  // FormData 不强制 Content-Type，让浏览器自动带 boundary
  if (!(options.body instanceof FormData)) {
    headers['Content-Type'] = 'application/json';
  }

  const res = await fetch(url, { ...options, headers });
  if (!res.ok) throw new ApiError(`HTTP ${res.status}`, res.status);
  const json = await res.json();
  if (json.code !== 0) throw new ApiError(json.message || 'unknown error', json.code);
  return json.data;
}

class ApiError extends Error {
  constructor(message, code) { super(message); this.code = code; }
}
```

### 5.2 统一响应格式【必须】

所有非 SSE 接口返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": { /* 业务字段 */ }
}
```

- `code === 0` 表示成功，非 0 表示业务错误。
- HTTP 状态码仍按 REST 规范（200/4xx/5xx），HTTP 错误也走 `code !== 0` 路径。
- `message` 永远面向用户可读，前端可直接 toast 显示。
- `data` 字段缺失时返回 `{}`，不返回 `null`。

### 5.3 错误分级【必须】

| 错误类型 | 用户可见反馈 | 数据影响 |
|---|---|---|
| 网络断开 | toast "网络好像不太顺畅～" | 保留本地状态，不阻塞 UI |
| HTTP 5xx | toast "服务打盹了，稍后再试" | 同上 |
| HTTP 4xx | toast "请求出错了：{message}" | 同上 |
| 识菜失败 / 超时 | 弹窗内显示错误 + 重试按钮 | 不关闭弹窗 |
| AI 对话失败 | 替换占位消息为"AI 暂时没空，等下再问吧～" | 保留用户问题可重发 |
| 业务 `code !== 0` | toast 显示 `message` | 不写入 store |

文案口吻：不出现"系统"、"失败"等冰冷词，改为"打盹了"、"网络不太顺畅"。

### 5.4 loading 状态【必须】

每个 API 调用配套一个 loading ref，**由调用方持有**（不在 api.js 全局管理，避免过度设计）：

### 5.5 识图异步轮询封装【必须】

V2 关键变化：识菜改异步轮询（同步会因 VL 推理 5–15s 超时）。封装要求：

```javascript
// 轮询参数固定：1.5s 间隔，最多 20 次（30s 超时）
// 进度条前端模拟：0→80% 前 3s 线性，80% 后等待真实结果
async recognizeWithPolling(file, { onProgress, onStatus } = {}) {
  const { task_id } = await this.submitRecognize(file);
  for (let i = 0; i < 20; i++) {
    await sleep(1500);
    const r = await this.pollRecognize(task_id);
    onProgress && onProgress(Math.min(95, (i + 1) * 5));
    onStatus && onStatus(r.status);
    if (r.status === 'done') return r.dish;
    if (r.status === 'failed') throw new ApiError(r.error || '识别失败');
  }
  throw new ApiError('识别超时');
}
```

- 取消：组件卸载时 `clearTimeout(pollingTimer)`，可选调 `POST /api/recognize/cancel/{task_id}`。
- 超时：30s 后提示"AI 有点忙，请稍后重试" + 重试按钮。

### 5.6 SSE 流式对话封装【必须】

AI 对话用 `fetch` + `ReadableStream` + `TextDecoder` 解析 SSE（非 `EventSource`，因 EventSource 不支持 POST）：

```javascript
/**
 * SSE 流式对话
 * @param {Object} payload - {persona, messages, context}
 * @param {{onDelta?: (s: string) => void, onMemoryHint?: (s: string) => void, onDone?: () => void, onError?: (e: Error) => void}} handlers
 */
chatStream(payload, { onDelta, onMemoryHint, onDone, onError }) {
  // 1. fetch POST /api/chat
  // 2. reader = res.body.getReader()
  // 3. TextDecoder 解 chunk
  // 4. 按 \n\n 切分事件，解析 data: 行
  // 5. type === 'delta' → onDelta(delta)
  // 6. type === 'memory_hint' → onMemoryHint(hint)
  // 7. [DONE] → onDone()
  // 8. 降级：SSE 不通时 POST /api/chat?stream=false 一次性返回
}
```

SSE 多事件类型契约：

```
data: {"type": "delta", "delta": "今"}
data: {"type": "delta", "delta": "天"}
data: {"type": "memory_hint", "hint": "你上周说胃不舒服"}
data: {"type": "delta", "delta": "还吃红烧鸡？"}
data: [DONE]
```

- 榜单头像：占位 emoji + IntersectionObserver 懒加载真实头像。

```javascript
// 正例
const io = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.dispatchEvent(new Event('visible'));
      io.unobserve(entry.target);
    }
  });
}, { rootMargin: '100px' });
```

### 10.4 防抖 debounce【必须】

| 场景 | 防抖时间 | 说明 |
|---|---|---|
| AI 文本更新 | 300ms | 现有 `main.js` 第 535 行已实现，保留 |
| 推荐请求 | 3 秒 | 菜盘变化后 debounce |
| 搜索 | 不需要 | 现有 `filteredFoods` 是 computed |
| 发送对话 | 不需要 | 用 `chatSending` flag 防连击 |

```javascript
/**
 * 防抖
 * @param {Function} fn
 * @param {number} delay
 * @returns {Function}
 */
function debounce(fn, delay) {
  let timer = null;
  return function (...args) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
```

### 10.5 节流 throttle【建议】

- 拼饭列表刷新：5 秒轮询（或 SSE，后端有余力时升级）。
- 时间更新：现有 `setInterval(updateTime, 60000)` 为正例。
- 滚动事件：节流 100ms。

### 10.6 并发优化【必须】

```javascript
// 正例：三者并发，不串行 await
async function onRecognizeDone(dish) {
  addFood(dish);                                    // 本地同步
  await Promise.all([
    api.getRecommend(currentMode.value, trayIds),   // 推荐请求
    updateAiText(false),                            // AI 文案更新
  ]);
}

// 反例：串行 await
async function onRecognizeDone(dish) {
  addFood(dish);
  await api.getRecommend(currentMode.value, trayIds);
  await updateAiText(false);
}
```

### 10.7 缓存【建议】

| 数据 | 缓存时长 | 失效条件 |
|---|---|---|
| 菜单数据 | 5 分钟 | TTL 过期 / 切换餐次 |
| 推荐结果 | 1 分钟 | 菜盘变化即失效 |
| B 端概览 | 1 分钟 | TTL 过期 |
| `FOODS_DATA` | 永久 | 常量，不变 |

### 10.8 PWA Service Worker 缓存策略【P3】

- 缓存 `index.html` / `style.css` / `main.js` / `constants.js` / `FOODS_DATA`。
- AI 调用走 **network-first**，失败回退缓存。
- 静态资源走 **cache-first**。
- 离线时显示"离线模式，仅可查看菜盘 + 本地菜品库"。
- 不支持 Service Worker 的浏览器（旧 iOS）正常使用在线模式，不报错。

### 10.9 不做的事【必须】

- 不做虚拟列表（菜品数 < 100 不需要）。

## 12. 错误处理规范

### 12.1 前端 fallback【必须】

每个异步操作必须有 fallback：

| 场景 | 失败 fallback |
|---|---|
| AI 文案 | 回退到本地 `generateAiText()`（现有正例，`main.js` 第 222–249 行） |
| 识菜 | 弹窗内显示错误 + 重试按钮，不关闭弹窗 |
| AI 对话 | 替换占位消息为"AI 暂时没空，等下再问吧～"，保留用户问题可重发 |
| 推荐补菜 | 推荐卡不显示，不阻塞主流程 |
| 菜单页 | 显示"菜单拉取失败，下拉重试" |
| 周报页 | ECharts 不渲染，显示文字版周报 |
| 拼饭列表 | 显示"暂无拼饭，下拉刷新" |

### 12.2 后端不通降级【必须】

- 网络断开：toast "网络好像不太顺畅～"，保留本地状态。
- HTTP 5xx：toast "服务打盹了，稍后再试"。
- SSE 不通：降级为非流式 `POST /api/chat?stream=false`。
- 识菜超时（> 30s）：提示"AI 有点忙，请稍后重试" + 重试按钮。

### 12.3 白屏预防【必须】

- `index.html` 的 `#app` 内必须有静态兜底内容（loading 占位），Vue 加载失败时仍可见。
- Vue 加载失败的检测：在 `index.html` 末尾加超时检查：

```html
<script>
  // Vue 加载超时兜底
  window.addEventListener('load', () => {
    setTimeout(() => {
      if (!window.Vue) {
        document.getElementById('app').innerHTML =
          '<div style="padding:40px;text-align:center;">页面加载失败，请检查网络后刷新</div>';
      }
    }, 5000);
  });
</script>
```

- 所有 `JSON.parse(localStorage.getItem(...))` 必须 try/catch 或加 `|| '[]'` 兜底（现有 `main.js` 第 265 行为正例）。
- 模板内访问嵌套属性用可选链 `?.`（现有 `categoryEmoji(cat) { return CATEGORY_META[cat]?.icon || ICONS.staple; }` 为正例）。

### 12.4 错误上报【建议】

- 关键错误（白屏 / API 5xx 频发）建议上报后端 `POST /api/log`，但**不阻塞用户操作**。
- 量级 <1 万，不做完整监控体系，仅记录关键错误。

## 13. 安全规范

### 13.1 XSS 防护【必须】

- 用户输入（菜名 / 对话 / 评价 / 备注）渲染前必须转义。
- Vue 模板默认 `{{ }}` 双花括号会自动转义，**禁止用 `v-html` 渲染用户输入**。
- 现有 `v-html="aiIcon"` / `v-html="categoryEmoji(...)"` 渲染的是**静态 SVG 常量**（`ICONS`），不是用户输入，可保留。
- 用户输入的菜名等用 `{{ }}` 渲染（现有 `{{ item.name }}` 为正例）。

```html
<!-- 正例：用户输入用 {{ }} -->
<span class="tag-name">{{ item.name }}</span>

<!-- 反例：用户输入用 v-html -->
<span class="tag-name" v-html="item.name"></span>
```

### 13.2 API key 不进前端【必须】

- 通义千问 VL API key + 阿里百练 API key 都**不进前端代码、不进 localStorage、不进环境变量**。
- 前端只调自己的 FastAPI 接口（`/api/recognize-dish` / `/api/chat` 等），由后端持有 key 转发到对应平台。
- 部署时 key 通过环境变量注入后端，不写进配置文件仓库。
- Code review 必须检查前端代码无 `sk-` / `AK` / `api_key` 等敏感字段。

### 13.3 CORS 配合【必须】

后端 FastAPI 必须配置 CORS：

```python
# 后端示意（非前端代码）
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5500", "http://127.0.0.1:5500"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

注意：

- `allow_origins` 必须包含前端实际访问的 host（包括 IP 形式，真机调试时手机访问开发机 IP）。
- SSE 流式响应必须确认 nginx / 中间件不缓冲响应（`X-Accel-Buffering: no` 头）。
- 识菜 multipart 上传需调大 `max_upload_size`（建议 10MB）。
- 异步轮询接口每次请求耗时 < 50ms，避免轮询本身拖慢。

### 13.4 Token 存储【建议】

- `anon_token` / `admin_token` 存 localStorage（量级小，无敏感数据）。
- token 由后端下发，前端不生成。
- admin 接口走单独 admin token，与 C 端 anonymous token 区分。

### 13.5 移动端真机调试注意【建议】

- 摄像头 API 需 `https` 才能调用（`capture="environment"` 在 http 下不生效，本地用 127.0.0.1 例外）。
- 语音 `webkit SpeechRecognition` 在 iOS Safari 部分版本不支持，需做特性检测。
- `js/api.js` 的 `API_BASE` 真机调试时不能写 `localhost`，必须写开发机 IP（手机访问 localhost 会指向手机自己）。
