# 食愈校园 — 前端 UI/UX 设计文档

> 版本：v1.0（基于 FRONTEND_DESIGN v2.0）
> 定位：站在 UI/UX 设计角度，定义三端（手机 / 平板 / 网页）视觉系统、组件形态、页面布局与交互规范
> 风格延续：治愈系手写风（暖橙 + 抹茶绿 + 米纸 + 虚线 + 手写字体）
> 输出目的：供产品评审、视觉走查、前端实现参考

## 1. 设计理念

### 1.1 V2 产品的情感定位

"食愈校园"不是工具，是"会成长、会记忆、会陪伴的校园饮食伙伴"。UI/UX 必须让用户在三端任何一屏上，都感到被理解、被照顾、被陪伴，而不是被一张冷冰冰的表格审视。

| 情感目标 | 设计落点 |
|---|---|
| 治愈 | 暖色调、圆角、纸张纹理、手写字体，避免锐利与冰冷 |
| 陪伴 | AI 搭子有人设头像、有口吻、有记忆气泡，主动 check-in |
| 成长 | 营养环、周报趋势图、记忆沉淀，让变化看得见 |
| 校园感 | 食堂场景、同学搭配榜、拼饭，氛围像下课后的食堂 |

### 1.2 三条不可妥协的设计原则

1. **治愈系视觉不可稀释**：新增三端布局、新增辅助色，但不替换主色板、不替换字体、不替换虚线 + 圆角 + 纸张纹理这套"治愈系 DNA"。新组件必须能在现有 phone-frame 内通过视觉走查。
2. **手机端是身份，不是降级**：手机端保留 phone-frame 手机外壳模拟，是产品"治愈系手写风"的身份来源；网页端不是手机端的放大，而是同一种治愈语言在不同尺寸上的自然展开。
3. **三端同一份情感**：无论用户在 375px 手机还是 1440px 桌面上看"AI 搭子记得你说过胃不舒服"，那种被想起的温暖感必须一致。布局可以变，温度不能变。

### 1.3 情感化设计四原则（贯穿全文档）

| 原则 | 含义 | 在本项目落点 |
|---|---|---|
| 柔和感 Softness | 圆角、柔光、低饱和度，避免锐角与硬边 | 圆角统一 12-24px，阴影多层柔光，色板饱和度克制 |
| 呼吸感 Rhythm | 周期性微动画 + 留白节奏，给视觉喘息空间 | 加载用呼吸环而非进度条，列表项错峰入场，间距 1.5x 节奏 |
| 反馈感 Responsiveness | 即时且克制的动效响应用户操作 | 按钮弹性回弹、添加成功 celebrate、SSE 打字光标 |
| 安全感 Safety | 不让用户犯错、不惩罚错误、错误时温柔提示 | 忌口拦截确认、识菜低置信度提示核对、错误文案"打盹了"而非"失败" |

## 2. 网络灵感参考

### 2.1 Smart Nutrition Tracker & Meal Planner App（Dribbble）

链接：https://dribbble.com/shots/26885895-Smart-Nutrition-Tracker-Meal-Planner-App

设计亮点：
- 暖橙色调（#E7360C / #C5A490）+ 软渐变 + 圆角组件，营造温暖与动力并存的体验。
- 主页中央圆形进度环展示每日卡路里目标 / 已摄入 / 已消耗 / 剩余，一眼可读。
- 餐食推荐卡片按餐次分类，配图 + 卡路里区间，降低决策疲劳。
- 整体视觉语言"清晰、平衡、情感舒适"，专为长期习惯养成设计。

借鉴点：
- 圆形进度环 = 本项目营养环的直接对标，可借鉴"中心放总数 + 外环分三段营养"的视觉结构。
- 暖橙 + 软渐变 = 本项目主色板已含 #F97316，可在 AI 气泡、按钮上加暖橙渐变提升温度。
- 按餐次分类的推荐卡片 = 本项目餐次卡 + 推荐补菜卡的结构参考。

### 2.2 大学生轻断食 APP "悦断食"（学术设计实践）

借鉴点：
- "科学-艺术-社交"三维模型 = 本项目"看见-懂得-陪伴-连接"四大支柱的镜像，可借鉴其"专业数据 + 审美界面 + 校园社交"的产品结构思路。
- 课表联动思路 = 本项目餐次时间提醒可借鉴"按作息灵活调整"的设计，而非固定时间。
- 校园社交强化动力 = 本项目榜单 / 拼饭的"匿名 + 同龄人搭配"提供了理论支撑。

### 2.3 Lovera: AI Companion（情感陪伴 App）

链接：https://www.trendebut.com/lovera-ai-companion-detailed-review-and-download-guide-1775791905

设计亮点：
- 镇定色板营造平静氛围，专为情感陪伴场景设计。
- 记忆功能：AI 记住过往对话、情绪模式，建立深度连接。
- 主动 check-in：AI 主动发起关怀，而非等用户开口。
- 隐私安全：所有对话保密，提供安全表达空间。

借鉴点：
- 镇定色板 = 本项目 AI 对话区可引入浅蓝（记忆气泡 #93C5FD）作为"冷静陪伴色"，与暖黄主气泡形成冷暖对偶。
- 记忆功能 = 本项目"会记忆的 AI 搭子"的核心差异化，记忆气泡的视觉形态可借鉴其"克制、不喧宾夺主"的设计。
- 主动 check-in = 本项目餐次提醒 + 情绪日记 + AI 主动提起过往的设计参考。

### 2.4 响应式设计最佳实践 2026（UXPin）

链接：https://www.uxpin.com/studio/blog/best-practices-examples-of-excellent-responsive-design/

设计亮点：
- Container Queries（2026 标准）：组件根据自身可用空间自适应，而非仅看视口。
- Fluid Typography 用 `clamp()`：字号随视口平滑缩放，无突变。
- Mobile-first 设计：从最小屏开始，渐进增强。
- 触控热区 ≥ 44×44px。

借鉴点：
- 三端断点设计：本项目从手机 375 起步，平板 768、网页 1024 三段断点。
- 触控热区：本项目手机端按钮、tab、菜品卡片必须 ≥ 44px 高。
- 流式排版：标题字号用 `clamp()` 在三端平滑过渡，而非三个断点跳变。
- 组件优先于页面：组件设计必须能独立适配三端，而非每页重写。

### 2.5 Flutter AI 聊天应用升级版（流式打字 + 欢迎屏）

链接：https://blog.csdn.net/2501_93573294/article/details/157616164

设计亮点：
- 欢迎屏：首次进入显示 AI 头像 + 引导副标题 + 4 个快捷提示词卡片，降低首次交互门槛。
- 流式响应：每 30ms 输出一字符，模拟真人打字，提升真实感与期待感。
- 动态闪烁光标：流式输出时末尾出现淡入淡出光标，赋予 AI"生命感"。
- 时间戳：消息外侧显示时间，不干扰正文。

借鉴点：
- 欢迎屏 = 本项目首次进入人设选择 modal + AI 欢迎语的设计参考。
- 流式打字 = 本项目 SSE 打字机光标的直接对标，已沿用现有 CSS。
- 快捷提示词卡片 = 本项目 AI 对话输入区上方可加"这餐蛋白够吗 / 帮我搭三餐"快捷气泡。

### 2.6 零点校园外卖（校园餐饮场景信息架构）

链接：https://www.0xiao.com/news/84134.html

设计亮点：
- 卡片式设计：每张外卖卡片如杂志内页，高清图 + 折叠菜单 + 渐变折扣标签。
- 场景化入口：按早餐 / 午市 / 夜宵划分时段，解决"不知道吃什么"。
- 进度条情绪管理：配送倒计时用舒缓渐变色，超时转温和提醒而非刺眼警报。
- 无中断加购：浏览推荐流时可侧滑调出迷你购物车，主界面保持 90% 可见。

借鉴点：
- 场景化入口 = 本项目餐次卡按早午晚分时段的设计已沿用。
- 进度条情绪管理 = 本项目识菜异步轮询进度条的设计参考——用舒缓色而非焦虑红。
- 无中断加购 = 本项目"加菜 + 不离开菜单页"的交互参考。

## 3. 三端响应式策略

### 3.1 断点定义

| 端 | 断点 | 视口宽度 | 设计基准 | 布局策略 |
|---|---|---|---|---|
| 手机端 | `sm` 及以下 | < 768px | 375×812 | 保留 phone-frame 外壳，单列布局 |
| 平板端 | `md` | 768px - 1023px | 768 / 820 | 退 phone-frame 外壳，自适应布局，可左右分栏 |
| 网页端 | `lg` 及以上 | ≥ 1024px | 1280 / 1440 | 桌面布局，左导航 + 主内容 + 右侧辅助栏 |

断点遵循 Mobile-first：默认写手机端样式，用 `@media (min-width: 768px)` 和 `@media (min-width: 1024px)` 渐进增强。

### 3.2 每端布局策略

- 平板竖屏（768×1024）默认此布局；横屏（1024×768）可考虑三栏（见网页端）。
- 弹窗 / modal 仍居中全屏，但宽度自适应。

### 3.3 组件在不同断点下的形态变化

| 组件 | 手机端 | 平板端 | 网页端 |

| B 端后台 | 不支持（强制转桌面） | 简化版 | 完整三栏桌面 |

## 4. 视觉系统

### 4.1 色彩系统

延续现有色板，新增辅助色以承载 V2 的情感维度（记忆 / 情绪 / 多模态）。

## 7. 交互设计原则

### 7.1 反馈及时性

| 操作 | 反馈 | 时长 |
|---|---|---|
| 按钮点击 | 弹性回弹 + ripple | 100ms |
| 加菜成功 | Toast"✓ 已添加" + tagPop | 即时 |
| 营养达标 | celebrate 动画 + Toast | 500ms |
| AI 流式 | 打字光标 + 文本追加 | 30ms/字 |
| 识菜提交 | 进入 loading 态 + 进度条 | 即时 |
| 网络异常 | Toast"网络不太顺畅～" | 即时 |

### 7.2 操作可见性

- 所有可点击元素 hover / active 必须有视觉变化（阴影 / 上浮 / 颜色）。
- 加载态必显示（呼吸环或进度条），不让用户以为卡死。
- 禁用态必灰显 + cursor not-allowed，不让用户误点。
- 长操作必可取消（识菜轮询可取消、AI 对话可中断）。

### 7.3 错误预防

| 场景 | 预防措施 |
|---|---|
| 添加过敏菜品 | 弹"⚠️ 这道菜含 XX，确认要加吗？"拦截 |
| 识菜低置信度 | "🤔 不太确定，请核对"提示用户复核 |
| 删除菜品 | 无需二次确认（可撤销 Toast） |
| 退出编辑 | 未保存时弹"确定放弃修改？" |
| 网络断开 | 顶部条显示"离线模式" + 禁用 AI 操作 |

### 7.4 情感化微交互

| 场景 | 微交互 |
|---|---|
| 首次选人设 | 人设头像 persona-wave 打招呼 |
| 情绪 emoji 选中 | mood-ripple 涟漪扩散 |
| 记忆气泡出现 | memory-hint-float 上浮淡入 |
| AI 流式打字 | 打字光标闪烁 + 文本逐字追加 |
| 营养达标 | celebrate 一次性庆祝动画 |
| 添加菜品 | tagPop 弹入 |
| 切换 tab | slide 过渡 |
| 拼饭有人加入 | 卡片轻微脉冲 + Toast |

## 8. 可访问性设计

### 8.1 键盘导航

- 所有交互元素必须可 Tab 聚焦，焦点可见（暖橙 outline 2px）。
- 弹窗打开后焦点锁定在弹窗内，关闭后焦点回到触发元素。
- Esc 键关闭弹窗。
- Enter 键提交输入（AI 对话、搜索）。

### 8.2 ARIA 标注

| 组件 | ARIA |
|---|---|
| 弹窗 | `role="dialog"` `aria-modal="true"` `aria-labelledby` |
| tab 切换 | `role="tablist"` `role="tab"` `aria-selected` |
| 营养环 | `role="img"` `aria-label="今日 kcal 1800，已达标"` |
| AI 对话历史 | `role="log"` `aria-live="polite"`（流式时朗读） |
| Toast | `role="status"` `aria-live="polite"` |
| 加载态 | `aria-busy="true"` |
| 图标按钮 | `aria-label` 描述用途 |

### 8.3 对比度

| 元素 | 对比度要求 | 检查 |
|---|---|---|
| 正文（暖灰字 #57534E on 米纸 #FEF9F0） | ≥ 4.5:1 | 通过 |
| 次要文字（暖灰 #A8A29E on 白卡） | ≥ 3:1 | 通过 |
| 按钮白字 on 暖橙 #F97316 | ≥ 4.5:1 | 通过 |
| 记忆气泡蓝字 on 浅蓝 10% | ≥ 4.5:1 | 需校验 |

治愈系色板饱和度低，需重点校验辅助色对比度。

### 8.4 prefers-reduced-motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

关闭所有非必要动画（celebrate、tagPop、呼吸环、打字光标），保留即时反馈（按钮按下）。

### 8.5 触控热区

- 手机端所有可点击元素 ≥ 44×44px。
- tab 项高度 ≥ 44px。
- 列表项内删除 ✕ 按钮热区扩展到整行右侧。

### 8.6 字号

- 正文最小 14px，避免移动端模糊。
- 输入框 font-size 14px+，防 iOS 缩放。
- 数字字号大（Number-L 32px+），保证营养环中心数字清晰可读。

## 9. 动效设计

### 9.1 入场动效

| 元素 | 动效 | 时长 | 缓动 |
|---|---|---|---|
| 卡片 | fade-in-up（上浮 20px + 淡入） | 300ms | ease-out |
| 列表项 | 错峰 fade-in-up（每项 delay 0.05s） | 300ms | ease-out |
| 弹窗 | slide-up（手机）/ fade + scale（平板网页） | 300ms | cubic-bezier(0,0,0.2,1) |
| 页面切换 | fade + slide（Vue transition） | 500ms | cubic-bezier(0,0,0.2,1) |

### 9.2 过渡动效

| 场景 | 动效 | 时长 |
|---|---|---|
| tab 切换 | 内容 fade 200ms | 200ms |
| hover | 上浮 2px + 阴影加深 | 200ms |
| modal 打开 | 遮罩 fade + 弹窗 scale 0.95→1 | 300ms |
| modal 关闭 | 反向，200ms | 200ms |

### 9.3 达标庆祝动效 celebrate

| 阶段 | 动效 | 时长 |
|---|---|---|
| 触发 | 营养环点亮 + 绿色 `--shadow-green` | 即时 |
| 庆祝 | 环外扩散涟漪 + 飘出 emoji（🎉 / ⭐） | 800ms |
| 收尾 | emoji 上浮淡出 | 400ms |

一次性播放，不循环。

### 9.4 记忆提示出现 memory-hint-float

| 阶段 | 动效 | 时长 |
|---|---|---|
| 出现 | 从气泡位置上浮 8px + 淡入 | 300ms |
| 停留 | 静态展示 | 默认 |
| 消失 | 淡出（如用户关闭） | 200ms |

### 9.5 SSE 打字动效

| 阶段 | 动效 | 时长 |
|---|---|---|
| 流式 | 文本逐字追加 + 末尾打字光标闪烁 | 30ms/字 |
| 光标 | 淡入淡出 500ms 周期 | 500ms |
| 结束 | 光标淡出 + 自动滚动到底 | 200ms |

降级：`prefers-reduced-motion` 关闭光标闪烁，直接显示完整文本。

## 10. 图标与插画规范

### 10.1 图标风格

沿用现有 SVG 风格：
- 线性图标（stroke），2px 描边
- 圆角线帽（stroke-linecap: round）
- 颜色用 CSS 变量（currentColor），继承父级
- 尺寸 16px（小）/ 20px（中）/ 24px（大）

### 10.2 emoji 使用

| 场景 | emoji | 说明 |
|---|---|---|
| 餐次 | 🌅 早 / ☀️ 午 / 🌙 晚 / 🍪 加餐 | 餐次卡标题 |
| 食物分类 | 🍚 主食 / 🍖 肉类 / 🥗 蔬菜 / 🥚 蛋奶 / 🍲 汤品 / 🍎 水果 | 菜品标签 |
| 人设 | 👩‍🍳 阿姨 / 🧑‍🎓 学长 / 👩‍🎓 学姐 | AI 搭子头像 |
| 情绪 | 😀 😢 😡 😴 🥰 | 情绪日记 |
| 功能 | 📷 拍照 / 🎙️ 语音 / 🍱 截图 / 📅 周报 / 🏆 榜单 / 🍚 拼饭 / 💬 评价 | 功能入口 |
| 提示 | 💡 推荐 / 🧠 记忆 / ⚠️ 警告 / ✅ 成功 | 状态提示 |

### 10.3 插画规范

- 空态插画：手绘风 SVG，暖色调，避免写实。
- 加载插画：呼吸环 + 暖色渐变。
- 错误插画：温和的"打盹"形象，不用刺眼红色 ❌。

### 10.4 头像规范

- 人设头像：emoji + 圆形底（暖黄 10% 背景）
- 用户头像：默认占位 emoji + 圆形底
- 榜单头像：占位 emoji + Intersection Observer 懒加载
